# v1.74.6-beta — Security: Zip-Slip + `require_admin` / `require_operator` proxmox-auth bypass (GitHub #4)

Two confirmed vulnerabilities reported via GitHub issue
[#4](https://github.com/P3Portal-org/p3portal/issues/4) by
[@cutetux](https://github.com/cutetux):

## Fixes

- **Zip-Slip in `save_playbook_zip()`** — the existing `..` reject
  loop was correct but did not cover entries like
  `/etc/cron.d/<file>`. `pathlib`'s join discards the left-hand path
  when the right-hand side is absolute, so a crafted ZIP could write
  outside the playbook directory. Closed with absolute-path
  rejection plus a `resolve().is_relative_to()` containment check.
- **`require_admin` / `require_operator` bypass for
  `auth_type="proxmox"`** — both dependencies short-circuited on any
  Proxmox-authenticated user regardless of portal role, opening the
  admin-only ZIP upload path to every Proxmox login. The
  short-circuit has been removed; portal role checks now apply
  uniformly to all auth types. (Note: the Proxmox cookie-auth path
  itself is deprecated — see
  [`docs/proxmox-setup.md`](https://github.com/P3Portal-org/p3portal/blob/main/docs/proxmox-setup.md)
  — but the bypass was still reachable and is now closed.)

Regression tests added under `backend/services/test_playbook_service.py`.
