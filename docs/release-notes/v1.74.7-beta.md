# v1.74.7-beta — Packer Zip-Slip defense-in-depth + `require_admin_or` proxmox-auth bypass closure

Second-wave hardening on top of v1.74.6-beta, applying the same fixes
that landed for the playbook upload path to the Packer upload path as
well, plus closing a related bypass in `require_admin_or`.

## Changes

- **`backend/services/packer_service.py`** — Zip-Slip hardening on
  `save_template_zip()`: reject ZIP entries with absolute paths (Unix
  / Windows) before joining onto `template_dir`, plus a
  defense-in-depth check that the resolved destination stays inside
  the template directory.
- **`backend/services/test_packer_service.py`** (new) — regression
  tests covering absolute paths (POSIX + Windows backslash form),
  `..` traversal, and a positive-case test.
- **`backend/core/deps.py`** — `require_admin_or()` no longer
  bypasses the role / permission check for `auth_type == "proxmox"`
  users. Proxmox-authenticated users now go through the same portal
  RBAC pipeline as local users, closing the second half of the
  GitHub #4 bug class (the first half landed in v1.74.6-beta).
- **`SECURITY.md`** — full overhaul: GitHub Private Vulnerability
  Reporting for security issues, public GitHub Issues for regular
  bugs, single-maintainer / LAN-only / archived-project clauses,
  full liability disclaimer, 90-day non-binding disclosure window.
